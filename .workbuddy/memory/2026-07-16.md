### 2026-07-16 追加：修复解析 bug — kills/damage 全为 0 + mode 为 null (11:33)
**问题：** 战报所有玩家 kills=0, damage=0, mode=null。两个根因：
1. **mode 正则不匹配 `_(Normal)_` 格式**：原 regex 用 `[_\(（]`（单字符），但实际文本是 `_(Normal)_`，在下划线后又紧跟括号，导致匹配失败 → mode 全部为 null
2. **kills/damage 值在同一行**：WebFetch 返回 `Kills 2`（同行），但代码按两行解析 (`line == "Kills"` 然后取下一行) → kills/damage 永远为 0

**修复：**
- mode regex: `[_\(（]` → `[_\(（]*`，`[_\)）]` → `[_\)）]*`（零或多个分隔符）
- kills/damage/dbnos/Traveled/Time Alive/Longest: 改为 `line.startswith("Kills")` 后从同行提取值，兼容同行和跨行两种格式
- 重新生成战报，数据正确：照诚 27杀/4067伤, 旺仔 19杀/2859伤, 仲阳 15杀/2379伤, 小明 9杀/1690伤, 老付 6杀/753伤
